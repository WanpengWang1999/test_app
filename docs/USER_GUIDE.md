# 工程照片采集系统使用说明

## 1. 系统用途

本系统用于通信工程建设和验收阶段的设备照片采集、自动命名、离线同步、进度检查和成果导出。现场人员按“项目 → 任务点 → 设备位 → 照片类型”逐级进入拍照，系统保存原图和水印图。

正式云端地址：

```text
http://114.55.109.150:3001
```

Android App 已固定连接该地址。

## 2. 账号和权限

| 角色 | 可用功能 |
| --- | --- |
| 完全管理员 | 管理全部项目、账号、导入、导出、健康页、备份恢复和数据清理。 |
| 项目管理员 | 管理全部项目，可新建项目、导入 Excel、维护照片类型、导出成果、审批和管理采集员。 |
| 采集员 | 选择项目采集照片、查看进度、同步本机队列、删除并重拍自己上传的照片。 |

新用户在登录页点击“注册账号”，填写手机号、姓名、工作单位和密码。注册后默认为采集员，必须由管理员审批通过后才能登录。

## 3. 登录

- 正式登录使用手机号和密码。
- 旧测试账号暂时兼容用户名登录。
- App 登录成功后会保存登录状态，下次打开自动进入系统。
- 如果账号待审批、被拒绝、停用或登录过期，系统会回到登录页。

## 4. Excel 导入

管理员进入“更多 → 导入 Excel”。

推荐先下载输入模板，字段包括：

| 字段 | 说明 |
| --- | --- |
| 项目 | 项目名称，系统从 Excel 读取。 |
| 任务点 | 站点、楼层、区域或施工点名称。 |
| 设备位 | 具体设备位置或设备名称。 |
| 设备编号 | 可选，用于辅助搜索和识别。 |
| 设备类型 | 例如 RRU、BBU、天线、通用。 |

上传后系统会先预检查，显示项目名称、新增任务点、新增设备、重复项和问题行，确认后再导入。

## 5. 现场采集流程

1. 登录后进入“项目”页面。
2. 点击项目的“进入采集”。
3. 在任务点列表搜索并进入任务点。
4. 在设备列表搜索并进入设备位。
5. 选择照片类型并拍照。
6. 拍照后先进入确认页，可取消、重拍或确认加入同步队列。
7. 必拍类型拍完后进入“额外拍摄照片”，并一直停留在额外拍摄照片。

如果现场发现 Excel 中没有任务点或设备位，所有角色都可以临时新增并继续拍照，后续由管理员检查补录项。

## 6. 同步中心

同步中心管理当前手机或当前浏览器自己的本地离线队列。

- 按项目、任务点查看待同步照片。
- 网络恢复后自动同步。
- 失败照片显示具体原因。
- 支持单张重试、全部重试和删除本地队列照片。

## 7. 采集进度

进度页按层级查看：

1. 项目总进度。
2. 任务点进度。
3. 设备进度。
4. 照片明细。

每层都支持搜索，方便项目数量、任务点数量和设备数量较多时定位。

## 8. 照片保存和命名

新上传照片按以下规则命名：

```text
项目名-任务点名-设备名称-照片类型-序号.jpg
项目名-任务点名-设备名称-照片类型-序号_original.jpg
```

系统同时保存：

- 原图。
- 带水印图。

水印包含项目、任务点、设备、照片类型、拍摄时间、拍摄人，能获取 GPS 时也会写入 GPS。

## 9. 导出成果

管理员进入“更多 → 导出成果”。

导出前检查会显示：

- 缺拍设备。
- 质量提醒照片。
- 本机未同步照片。
- 回收站照片。
- 现场补录任务点和设备。

导出支持整个项目 ZIP，也支持按任务点单独导出 ZIP。

## 10. 回收站

删除照片不会立即物理删除，而是进入回收站。

- 采集员只能删除自己上传的照片。
- 管理员可恢复或彻底删除项目内照片。
- 回收站照片默认不进入导出成果。

## 11. 备份和健康检查

完全管理员进入“更多 → 健康与备份”。

支持：

- 查看后端、数据库、上传目录、数据目录和版本信息。
- 一键备份 SQLite 数据库和 uploads 照片目录。
- 查看最近备份。
- 上传备份恢复。

## 12. 常见问题

### 新注册账号不能登录

账号需要管理员审批。审批通过前登录会提示“账号待管理员审批后才能登录”。

### App 无法连接服务器

确认手机可以访问：

```text
http://114.55.109.150:3001/api/public/health
```

如果无法访问，检查域名解析、HTTPS 证书、Nginx、服务器安全组和后端服务。

### 照片没有同步

进入“同步”页面查看失败原因。常见原因包括网络断开、账号登录过期、服务器不可达或照片过大。
