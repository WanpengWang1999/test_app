# 工程照片采集系统使用说明

## 1. 系统用途

本系统用于通信工程建设和验收阶段的设备照片采集、自动命名、同步、进度检查和成果导出。现场人员按“项目 → 任务点 → 设备位 → 照片类型”逐级进入拍照，系统保存原图和水印图，并生成 ZIP、Excel 清单和 JSON 清单。

## 2. 账号和权限

| 角色 | 可用功能 |
| --- | --- |
| 完全管理员 | 管理全部项目、账号、导入、导出、健康页、备份恢复、数据清理。 |
| 项目管理员 | 管理全部项目，可新建项目、导入 Excel、维护照片类型、导出成果、创建采集员。 |
| 采集员 | 选择项目采集照片、查看进度、同步本机队列、删除并重拍自己上传的照片。 |

采集员不能进入项目管理、Excel 导入、导出、账号、备份和数据清理页面。

## 3. 局域网部署与访问

1. 电脑和手机连接同一个 Wi-Fi 或同一个局域网。
2. 在电脑项目目录运行 `start-lan.cmd`，或使用 PowerShell 执行：
   ```powershell
   powershell -ExecutionPolicy Bypass -File .\scripts\start-lan.ps1
   ```
3. 启动窗口会显示电脑访问地址、手机访问地址和后端地址。
4. 手机浏览器访问 `http://电脑局域网IP:5173/`。
5. Android App 首次打开时，在服务器地址中填写 `http://电脑局域网IP:3001`，点击测试连接后登录。

如果手机无法访问，优先检查：手机是否同网段、Windows 防火墙是否放行 Node.js、前端是否用 `0.0.0.0` 启动、后端端口是否为 `3001`。

## 4. Excel 导入

管理员进入“更多 → 导入 Excel”。

推荐先点击“下载输入模板”，模板字段包括：

| 字段 | 说明 |
| --- | --- |
| 项目 | 项目名称，系统会从 Excel 读取，不需要手填。 |
| 任务点 | 站点、楼层、区域或施工点名称。 |
| 设备位 | 具体设备位置或设备名称。 |
| 设备编号 | 可选，用于辅助搜索和识别。 |
| 设备类型 | 如 RRU、BBU、天线、通用。不同设备类型可配置不同必拍照片类型。 |

上传文件后系统会先预检查，显示项目名称、任务点数量、设备位数量、错误行、重复项和问题清单。确认无误后再点击“确认导入 Excel”。

## 5. 现场采集流程

1. 登录后默认进入“项目”页面。
2. 点击“继续上次采集”可回到最近一次项目、任务点和设备位。
3. 点击项目的“进入采集”。
4. 在任务点列表中搜索任务点，可勾选“只看未完成”。
5. 进入任务点后搜索设备位，可勾选“只看未完成”。
6. 进入设备位后选择照片类型并拍照。
7. 拍照后先进入确认页，可取消重拍或“确认并进入下一类型”。
8. 必拍类型拍完后，“下一种类型”会停留在“额外拍摄照片”，不会回到第一种类型。

如果现场发现 Excel 中没有任务点或设备位，所有角色都可以临时新增任务点或临时新增设备位并继续拍照。管理员后续可在导出检查中看到待确认补录。

## 6. 同步中心

同步中心管理的是当前手机或当前浏览器自己的本地离线队列。不同手机各自有自己的本地队列。

支持：

- 查看本机待同步数量、失败数量和占用空间。
- 按项目、任务点进入查看队列明细。
- 网络恢复后自动同步。
- 失败照片显示具体错误，可单张重试或全部重试。
- 不想上传的本地队列照片可删除。

## 7. 采集进度

进度页按层级查看：

1. 项目总进度：已完成设备、总设备、待同步照片。
2. 任务点进度：每个任务点的完成数、同步数和待同步数。
3. 设备进度：查看设备缺失照片类型和已拍照片。

每一层都支持搜索，适合任务点和设备数量较多的项目。

## 8. 照片命名和保存

新上传照片按以下规则命名：

```text
项目名-任务点名-设备名称-照片类型-序号.jpg
项目名-任务点名-设备名称-照片类型-序号_original.jpg
```

序号在同一项目、同一任务点、同一设备、同一照片类型内递增。页面和导出清单按序号从大到小展示。

系统保留原图，同时上传带水印图片。水印包含项目、任务点、设备、照片类型、拍摄时间、拍摄人，能获取 GPS 时也会写入 GPS。

## 9. 导出成果

管理员进入“更多 → 导出成果”。

导出前检查会显示：

- 缺拍设备。
- 质量提醒照片。
- 本机未同步照片。
- 回收站照片。
- 现场补录任务点和设备。
- 待确认补录。
- GPS 缺失照片。

导出支持整个项目 ZIP，也支持按任务点单独导出 ZIP。项目 ZIP 内包含照片、Excel 清单和 JSON 清单。

## 10. 回收站

删除照片不会立即物理删除，而是进入回收站。

- 采集员只能删除自己上传的照片。
- 管理员可恢复或彻底删除项目内照片。
- 回收站保留期为 30 天。
- 导出默认不包含回收站照片。

## 11. 备份和恢复

完全管理员进入“更多 → 健康与备份”。

支持：

- 一键备份 SQLite 数据库和 uploads 照片目录。
- 查看最近备份列表。
- 下载备份 ZIP。
- 上传备份进行恢复。

恢复前系统会先生成当前安全备份。恢复完成后需要重启后端服务。

## 12. 诊断和 APK 更新

“更多 → 连接诊断”用于排查连接问题，显示：

- 后端连通状态。
- 登录状态。
- 浏览器网络状态。
- 本地同步队列数量。
- 上传目录是否可写。
- 数据目录、上传目录和局域网后端地址。
- 当前版本、构建时间和 APK 信息。

“更多 → 下载 APK”可下载当前电脑生成的 Android 内部测试 APK。如果提示尚未生成 APK，请先在电脑运行 Android 打包命令。

## 13. 常见问题

### 手机打不开网页

检查手机和电脑是否在同一局域网；确认访问的是 `http://电脑局域网IP:5173/`，不是 `localhost`；确认防火墙放行 Node.js。

### App 登录失败

确认 App 服务器地址填写的是后端地址，例如 `http://192.168.5.235:3001`，不是前端地址 `5173`。

### 拍照后没有立即上传

系统会先进入确认页。确认后照片进入本地队列；离线或弱网时会保留在同步中心，联网后自动同步。

### 照片处理失败

新版流程先保存原图和采集信息，上传时再生成水印图。如果仍失败，进入同步中心查看具体错误并重试。

### Excel 导入后设备类型照片类型不对

检查 Excel 的“设备类型”列是否填写一致，例如 `RRU`、`BBU`、`天线` 不要混用空格或不同写法。管理员可在导入页维护照片类型模板。
